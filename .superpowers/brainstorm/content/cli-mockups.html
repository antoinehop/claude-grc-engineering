1
Gap Assessment
One command pulls evidence from any connected source, crosswalks every finding against SCF's 1,468 controls, and produces a prioritized gap report across as many frameworks as you request simultaneously.
/grc-engineer:gap-assessment SCF crosswalk effort estimates
Auto-fix signal
Findings marked auto-fix can be remediated immediately with /grc-engineer:scan-iac --fix. No manual Terraform required.
Multi-framework in one pass
Passing SOC2,FedRAMP-High runs both frameworks against the same findings set. The SCF crosswalk means no double-collection.
2
IaC Scan + Auto-fix
Scans Terraform, CloudFormation, and Kubernetes manifests for compliance violations across any framework. The --fix flag writes remediations directly to the files — S3 encryption, log retention, KMS rotation, and more.
/grc-engineer:scan-iac auto-fix mode SARIF output
CI/CD integration
Pass --format=sarif to pipe results into GitHub Advanced Security or GitLab's security dashboard. Blocks PRs on critical findings.
What auto-fix covers
S3 encryption · S3 versioning · S3 public-access block · KMS key rotation · log retention periods. Security group rules and IAM policies are flagged but require manual review.

3
Cross-Framework Intelligence
Map any control to every framework it touches, surface conflicts between standards, and generate a single implementation that satisfies all of them. The average multi-framework engagement covers 815 controls — the crosswalk reduces that to 362 unique implementations.
/grc-engineer:map-controls-unified 56% effort reduction
Generate, don't write
After mapping, /generate-implementation outputs production-ready Terraform, a Python evidence collector, and a CloudWatch dashboard — all wired to the specific control.
Conflict resolution built in
When PCI DSS and NIST disagree on a requirement, the crosswalk surfaces the conflict and defaults to "most restrictive wins" — no manual reconciliation.
4
Continuous Monitoring
Schedule automated control tests on a daily or hourly cadence. Tracks 30-day compliance trends, detects degradation, and fires Slack or PagerDuty alerts before controls fail an audit.
/grc-engineer:monitor-continuous Slack · PagerDuty 30-day trends
Health indicators
Controls are classified as HEALTHY (>95% 30-day pass rate), DEGRADING (−5% in 7 days), or AT RISK (<80% pass rate).
Alert thresholds
Critical if overall compliance drops below 80%, or PCI DSS drops below 90%. Warning if any control degrades more than 5% in 7 days. Configurable via compliance-monitor.yaml.
